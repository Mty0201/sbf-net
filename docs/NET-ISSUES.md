## NET-01: 点密度自适应阈值

### 问题描述

sbf-net 制作 edge.npy 时使用固定参数：`sigma=0.04m`, `supervision_radius=0.08m` (2σ)。这些参数以绝对距离定义，不随局部点密度调整。

在稀疏区域（点间距 > 0.03m），0.08m 的 supervision_radius 仅覆盖极少数点，导致：
- 稀疏区 valid 边缘点仅占 9–14%（密集区 ~18%）
- 稀疏区 valid + weight ≥ 0.5 仅占 5–7%（密集区 ~10%）
- 经 grid thinning + outlier removal 后，稀疏区边缘参考点出现结构性空洞

### 对 recon 的影响

边缘参考点覆盖不足 → `relocate_boundary_vertices()` 在稀疏区 30% 的边界顶点找不到 snap 目标 → 毛边残留。recon 侧可通过自适应 snap_radius 和放宽离群剔除缓解（RECON R-01/R-02），但无法弥补上游数据本身的缺失。

### net 侧需要提供什么

**方案 A — 密度自适应 sigma:**
```
local_spacing = mean_knn_distance(coord, k=10)
sigma_local = max(sigma_base, alpha * local_spacing)  # e.g. alpha=1.5
supervision_radius_local = 2 * sigma_local
```
这使得稀疏区域的 supervision 范围自动扩大，覆盖更多点。weight 公式不变（仍为 Gaussian），只是分母随密度缩放。

**方案 B — 密度自适应 weight 阈值:**
在 edge.npy 中额外输出一列 `local_density` 或 `recommended_threshold`，让 recon 侧按密度分桶使用不同的 weight 阈值。

**推荐方案 A**，因为从源头增加覆盖比在下游调阈值更有效。

**数据格式变化:** 无。edge.npy 仍为 (N, 6)，只是稀疏区的 valid/weight 值会有更多非零项。

## NET-02: 单侧语义导致支撑元断开

### 问题描述

当边界一侧只有单一语义（如门洞边界，一侧是门面板，另一侧是空场景无点），sbf-net 的边界中心检测（Stage 1: 在 supervision_radius 内搜索不同语义标签的点）在该位置找不到跨语义对，导致：
- 该边界无边界中心 → 无 DBSCAN 聚类 → 无支撑元 → edge.npy `valid=0`
- 支撑元在此处断开，两侧有支撑元的区域正常工作

### 当前样本中的表现

020101/020102 中此问题不严重（所有边界对的覆盖 imbalance < 24%）。但在更复杂场景中（建筑入口门洞、阳台缺口、遮挡边缘）预期影响显著。

Type-2 分析显示：020102 有 12.3% 的密集边界点属于 dense+boundary+无 edge 的情况，集中在 seg 3（窗框 49%）和 seg 6（装饰线脚 46%）。

### 对 recon 的影响

无支撑元 → 无边缘参考点 → 该段边界完全无法 snap → 保留 Poisson 原始毛边。recon 侧只能做兜底平滑（RECON R-04），无法做精确的几何锐化。

### net 侧需要提供什么

**方案 A — 扩展边界检测的搜索半径:**
对于点云边缘/空洞区域，当单侧搜索不到异类点时，沿法线方向延伸搜索或使用几何边缘检测（如 sharp angle detection）代替纯语义检测。

**方案 B — 输出几何边缘置信度:**
在 edge.npy 中增加一列 `edge_type`（0=内部, 1=语义双侧, 2=语义单侧, 3=几何边缘），让 recon 针对不同类型选择策略。

**方案 C — 支撑元外推:**
在 supports.npz 中，对已有支撑元沿其走向做短距离外推（如 5× local spacing），填补因单侧语义导致的局部断开。

**推荐方案 A + C 组合。** A 解决检测问题，C 解决已有支撑元的连续性问题。

**数据格式变化:**
- 方案 A: 无变化（只是更多点获得 valid=1）
- 方案 B: edge.npy 扩展为 (N, 7)，增加 edge_type 列
- 方案 C: supports.npz 中增加外推标记

## NET-03: 支撑元抖动 / 蛇形线问题

### 问题描述

部分支撑元不是平直线段而是蛇形弯曲线。点投射到蛇形支撑元后，边缘参考点也呈蛇形分布。Tier-2 将边界顶点 snap 到蛇形参考点，边界从"Poisson 平滑毛边"变成"精确蛇形弯曲"——视觉效果反而更差。

### 量化证据

同段边缘参考点 snake score（局部 RMS 偏离拟合直线的距离）：

| 指标 | 020101 | 020102 |
|------|:------:|:------:|
| snake > 0.005m | 13.9% | 22.5% |
| snake > 0.01m | 4.4% | 10.8% |
| snake > 0.02m | 0.1% | 3.8% |

同段方向一致性（投射方向与局部均值方向的夹角）：

| 指标 | 020101 | 020102 |
|------|:------:|:------:|
| 偏差 > 10° | 77.6% | 77.5% |
| 偏差 > 20° | 55.3% | 55.5% |

方向高度不一致说明投射方向逐点变化大——这不是随机噪声（噪声会有各向同性分布），而是支撑元本身的几何导致每个点被拉向略有不同的方向。

### 蛇形的来源分析

**根因推测（需 net 侧确认）：**

1. **DBSCAN 聚类边界不规则:** Stage 2 聚类的边界本身就不是直线，聚类形状的不规则性传递到 Stage 3 支撑面拟合
2. **支撑面拟合采样分布不均:** 拟合平面时如果采样点在一侧集中，拟合面会向集中侧倾斜。沿边界走向，采样分布的变化导致拟合面法线逐点漂移
3. **多个支撑元交接处的不连续:** 两个相邻支撑元各自拟合良好，但交接处产生折线

### 对 recon 的影响

蛇形参考点使 Tier-2 的保边效果变差。recon 侧可以做链平滑缓解投射噪声（RECON R-03），但系统性弯曲无法在 recon 侧修复——平滑后的蛇仍然是蛇，只是更光滑。

020102 中 22.5% 的边缘参考点 snake > 0.005m，这意味着近四分之一的边界在 Tier-2 后会呈现可见的蛇形弯曲。

### net 侧需要提供什么

**方案 A — 支撑面拟合全局一致性约束:**
对同一边界上连续的支撑元，施加法线一致性约束（相邻支撑面法线差 < 阈值），抑制逐点的法线漂移。

**方案 B — 支撑线后处理平滑:**
Stage 3 拟合完成后，对支撑线/polyline 做 Laplacian 平滑或 B-spline 拟合，消除高频抖动。保留低频（真实曲率）去除高频（拟合噪声）。

**方案 C — 输出支撑元直线性置信度:**
在 edge.npy 或 supports.npz 中输出每个支撑元/每个点的 `linearity_score`（局部拟合残差），让 recon 侧对低置信度区域自动 fallback 到平滑策略。

**推荐方案 B（最直接有效）+ C（给 recon 决策依据）。**

**数据格式变化:**
- 方案 A/B: 无变化（支撑元更平直，edge.npy 中的 direction 自然更一致）
- 方案 C: edge.npy 扩展为 (N, 7) 增加 linearity_score 列，或在 supports.npz 中增加 per-support linearity 字段

## 问题关联与优先级

```
NET-01 (密度自适应) ──→ 缓解 RECON R-01/R-02（稀疏区覆盖不足）

NET-03 (蛇形支撑元) ──→ 根治 RECON R-03（蛇形参考点平滑只是缓解）
```

| 编号 | 问题 | 对 recon 质量的影响 | net 改动量 | 建议优先级 |
