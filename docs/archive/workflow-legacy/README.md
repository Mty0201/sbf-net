# Legacy Workflow Archive

This directory archives the manual workflow scaffolding that used to sit in the active repository path before the GSD migration.

Default workflow entry for current work is still:

1. `GSD`
2. `.planning/PROJECT.md`
3. `.planning/ROADMAP.md`
4. `.planning/STATE.md`
5. `docs/canonical/README.md`

Use this archive only when you need historical workflow context, old handoff material, old role/skill docs, or provenance for pre-GSD continuity artifacts.

## Archive Sections

- `ARCHIVE_MAP.md`: exact path-by-path migration map for active, archived, and compatibility surfaces
- `handoff/`: archived handoff and cross-window continuity material
- `project_memory/`: archived topical memory, task history, and pre-GSD continuity docs
- `codex/`: archived repo-local orchestration tooling, helper scripts, and mirrored Claude guides
- `wrappers/`: archived legacy entry docs and wrapper bodies that no longer belong in the active root path

## Read Order

1. Read `ARCHIVE_MAP.md` first to see where an old path moved.
2. Read the relevant archived subtree only if current GSD and canonical docs are insufficient.
3. Return to active surfaces for current decisions:
   - `AGENTS.md`
   - `docs/workflows/sbf_net_workflow_v1.md`
   - `docs/canonical/README.md`

## What Stays Active

These surfaces are not archived:

- `docs/canonical/README.md` and the canonical SBF technical guidance set
- `AGENTS.md`
- `docs/workflows/sbf_net_workflow_v1.md`
- `.planning/` and GSD commands
- reports used as evidence artifacts when needed

## Rule

Archived workflow material is historical reference only. It must not become the default control plane again.
