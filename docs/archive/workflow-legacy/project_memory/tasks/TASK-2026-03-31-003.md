# TASK-2026-03-31-003

## Goal

- 修正 full-train 分析链中的“验证口径与产物绑定”问题，使当前权威口径稳定收敛到 semantic scalar `val_mIoU`，并消除 summary / packet / workflow smoke / round artifact 之间的指标混淆与 primary summary conflict。

## Why now

- `TASK-2026-03-30-002` 已把当前 full-train 的主要问题收敛到“验证口径问题 + 记录/打包问题”为主，而不是模型主线或 loss 主线先失效。
- 若当前仍混用 step 级 `mIoU` 与 scalar `val_mIoU`，并且 packet / smoke / round draft 的 primary summary source 不一致，那么任何后续模型判断都会先被记录层噪声污染。
- 因此这一轮应先把验证与记录层修正到可解释、可复核，再决定是否需要进入模型实现轮。

## In scope

- 明确 trainer / evaluator 的验证指标命名与输出边界，确保 semantic 主权威指标口径清楚指向 scalar `val_mIoU`。
- 修正日志摘要层对 step 级指标与 scalar 级指标的区分与展示。
- 修正 context packet 对权威指标与 primary summary 的选取逻辑，避免把非权威 `mIoU` 混入主结论。
- 修正 workflow smoke / round artifact 对 primary summary source 的一致性检查与展示。
- 基于现有 full-train 日志和已生成摘要复核，不要求新训练。

## Out of scope

- 不修改模型主线、head、loss、data pipeline 或 Pointcept 主体。
- 不重新设计 `axis + side + support` 架构，不重开 `dist` 或其他旧主线。
- 不要求新训练、不追加新实验、不重跑 full train。
- 不扩展到 test/export/visualization 或跨多 task 历史整理。

## Constraints

- 唯一允许主动维护的项目根是 `sbf-net`。
- `Pointcept` 只是宿主依赖，默认只读；若问题疑似来自宿主接口，必须停止并汇报。
- 本轮是验证/记录层实现轮，不允许借题发挥去改模型、loss 或数据逻辑。
- 修正必须以“现有 full-train 日志可复核”为准，不能靠新训练掩盖口径问题。
- 结果中必须区分：
  - 权威指标口径
  - 展示层兼容信息
  - 仍待解释的模型层问题
