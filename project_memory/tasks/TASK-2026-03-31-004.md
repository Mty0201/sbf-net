# TASK-2026-03-31-004

## Goal

- 在已校准的权威口径下，分析为什么当前 full-train 的 best scalar `val_mIoU = 0.7316`、latest `val_mIoU = 0.7085` 低于 `support-only baseline = 74.6`，并据此决定下一轮应优先走调参重训、结构改动，还是补额外验证。

## Why now

- `TASK-2026-03-31-003` 已完成验证口径修正，workflow smoke 已达 `PASS`；当前 semantic scalar `val_mIoU` 已是可直接用于判断的权威指标。
- 在口径干净之前，任何“模型有问题”的结论都可能只是记录层误导；但现在口径已校准，真正的下一步应该是根因分析，而不是立刻进入实现。
- 若不先分析 full-train 低于 baseline 的原因，就会把“可学习性不足”“多任务拖拽”“优化失衡”“过拟合”这些不同问题混成同一种实现需求。

## In scope

- 基于现有 full-train 日志、摘要和必要的 checkpoint / best-epoch 记录，分析 semantic scalar `val_mIoU` 低于 baseline 的原因。
- 判断 direction / axis-side 分支是否存在可学习性问题，或是否对 semantic 主任务形成拖拽。
- 分析 support / dist 学到了什么、没学到什么，以及为什么没有转化成更高的 semantic `val_mIoU`。
- 判断后期退化更像过拟合、优化失衡，还是多任务拖拽。
- 最后给出下一轮优先路线：调参重训、结构改动，还是补额外验证。

## Out of scope

- 不在本轮修改业务代码、模型主线、loss 主线、head、data pipeline 或 Pointcept。
- 不在本轮直接执行调参重训或结构改动。
- 不要求新训练，不追加新实验。
- 不扩展到 test/export/visualization 或跨多 task 历史整理。

## Constraints

- 唯一允许主动维护的项目根是 `semantic-boundary-field`。
- `Pointcept` 只是宿主依赖，默认只读；若问题疑似来自宿主接口，必须停止并汇报。
- semantic scalar `val_mIoU` 是当前权威判断口径；step-level `val.mIoU` 不再作为主结论依据。
- 结果中必须区分：
  - 已确认事实
  - 待解释问题
  - 根因假设
  - 下一轮建议路线
- 若建议进入实现轮，必须先说明为什么“补验证”或“调参重训”不足以回答当前问题。
