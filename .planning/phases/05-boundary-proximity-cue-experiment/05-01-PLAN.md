# Plan 05-01: Implement Boundary Proximity Cue Loss and CR-C Config

## Goal

Implement the confidence-weighted BCE loss (Option L1 from route redesign), create the CR-C training config, and smoke-validate the full pipeline. This plan covers CUE-01 and CUE-02. CUE-03 (training) and CUE-04 (evaluation) are separate.

## Requirements Addressed

- **CUE-01**: Implement confidence-weighted BCE loss treating `valid` as binary boundary label and `support` as confidence weight
- **CUE-02**: Create CR-C training config identical to CR-B except loss and auxiliary weight

## Context

See `05-CONTEXT.md` for full design rationale and route redesign references.

**Key design from `docs/canonical/route_redesign_discussion.md` Section 5.3, Option L1:**
```
auxiliary_target = valid                          # binary: boundary or not
confidence_weight = where(valid, support, 1.0)    # Gaussian weight for boundary points, 1.0 for non-boundary
loss_aux = weighted_BCE(support_pred, valid, weight=confidence_weight)
```

**Key facts:**
- The model (`SharedBackboneSemanticSupportModel`) outputs `support_pred: (N, 1)` raw logits — sigmoid is applied in the loss, not in the model
- The trainer routes `support_pred` + `edge` to the loss via `_build_loss_inputs()` — no trainer changes needed
- Edge data format: `edge[:, 4]` = support (continuous, 0-1), `edge[:, 5]` = valid (binary)
- The existing `RedesignedSupportFocusEvaluator` accepts the same signature `(seg_logits, support_pred, segment, edge)` — no evaluator changes needed for Phase 5 (its SmoothL1/Tversky metrics will report on the same predictions, which is fine for tracking)

## Tasks

### Task 1: Implement `BoundaryProximityCueLoss`

**File:** `project/losses/boundary_proximity_cue_loss.py` (NEW)

**Class:** `BoundaryProximityCueLoss(nn.Module)`

**Constructor:**
```python
def __init__(
    self,
    aux_weight: float = 0.3,       # weight for auxiliary BCE term (0.2-0.5 range)
):
```

**forward() signature** (must match trainer routing — same as `RedesignedSupportFocusLoss`):
```python
def forward(
    self,
    seg_logits: torch.Tensor,     # (N, num_classes) from semantic head
    support_pred: torch.Tensor,   # (N, 1) raw logits from support head
    segment: torch.Tensor,        # (N,) ground-truth class labels
    edge: torch.Tensor,           # (N, 6) edge data
    **_extra,
) -> dict[str, torch.Tensor]:
```

**Loss computation:**
```python
segment = segment.reshape(-1).long()
edge = edge.float()

# Ground-truth extraction (same columns as RedesignedSupportFocusLoss)
support_gt = edge[:, 4].float().clamp(0.0, 1.0)
valid_gt = edge[:, 5].float().clamp(0.0, 1.0)

# === Term 1: Global semantic (CE + Lovasz) — identical to RedesignedSupportFocusLoss ===
loss_ce = self.semantic_loss(seg_logits, segment)
loss_lovasz = self.lovasz_loss(seg_logits, segment)
loss_semantic = loss_ce + loss_lovasz

# === Term 2: Confidence-weighted BCE (Option L1) ===
# Target: valid (binary: boundary or not)
# Weight: support value for boundary points, 1.0 for non-boundary points
support_logit = support_pred[:, 0]
aux_target = valid_gt                                        # binary boundary label
confidence_weight = torch.where(valid_gt > 0.5, support_gt, torch.ones_like(support_gt))

# BCE with logits for numerical stability
loss_aux_raw = F.binary_cross_entropy_with_logits(
    support_logit, aux_target, weight=confidence_weight, reduction="mean"
)

# === Total loss ===
total_loss = loss_semantic + self.aux_weight * loss_aux_raw
```

**Return dict:**
```python
return dict(
    loss=total_loss,
    loss_semantic=loss_semantic,
    loss_ce=loss_ce,
    loss_lovasz=loss_lovasz,
    loss_aux=loss_aux_raw,
    loss_aux_weighted=self.aux_weight * loss_aux_raw,
    # Monitoring metrics (no gradient)
    valid_ratio=valid_gt.mean(),
    support_positive_ratio=(support_gt > 1e-3).float().mean(),
    aux_prob_mean=torch.sigmoid(support_logit).mean().detach(),
    aux_prob_boundary_mean=(torch.sigmoid(support_logit) * valid_gt).sum().detach() / valid_gt.sum().clamp_min(1e-6),
)
```

**Design notes:**
- Uses `F.binary_cross_entropy_with_logits` (not `F.binary_cross_entropy`) for numerical stability — sigmoid is fused into the loss
- `confidence_weight` gives boundary points graduated importance (support=1.0 at boundary core, decay toward edges), and non-boundary points uniform weight 1.0
- The semantic loss (CE + Lovasz) is identical to CR-A and CR-B — no change
- No focus term, no Tversky, no SmoothL1, no regression

**Validation:** Forward pass with random inputs of correct shapes returns all expected dict keys with valid gradients.

## Risks

- **Low:** The `aux_weight=0.3` default may need tuning. The route redesign recommends 0.2-0.5. Starting at 0.3 is a reasonable midpoint. If monitoring shows aux gradient share > 20% by epoch 50, reduce in a follow-up config.
- **Low:** The evaluator reports SmoothL1-based `support_reg_error` for a model trained with BCE. The metric is still computable (it's just SmoothL1 between sigmoid output and support_gt*valid_gt), but its interpretation changes. This is acceptable for Phase 5 — the primary metric is val_mIoU, and boundary-region metrics are secondary.
- **None:** No Pointcept source modified. No model architecture changes. No data pipeline changes.
