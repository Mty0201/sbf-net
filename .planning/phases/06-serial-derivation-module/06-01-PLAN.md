### Task 1: Implement `BoundaryOffsetModule` in `project/models/heads.py`

**File:** `project/models/heads.py`

**Add new class** `BoundaryOffsetModule(nn.Module)`:

```python
class BoundaryOffsetModule(nn.Module):
    """Learnable module g: derives boundary offset field from semantic logits.

    Takes softmax(seg_logits) and coord, performs KNN local aggregation to
    capture local semantic gradients, and predicts a 3D offset vector per point.
    offset = displacement from current point to nearest boundary.
    """

    def __init__(self, num_classes, k=16, hidden_dim=64):
        super().__init__()
        self.k = k
        # Edge feature: [feat_i, feat_j - feat_i, coord_j - coord_i]
        # dim = (num_classes + 3) + (num_classes + 3) + 3 = 2 * num_classes + 9
        edge_feat_dim = 2 * num_classes + 9
        self.edge_mlp = nn.Sequential(
            nn.Linear(edge_feat_dim, hidden_dim),
            nn.ReLU(inplace=True),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(inplace=True),
        )
        self.offset_head = nn.Sequential(
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.ReLU(inplace=True),
            nn.Linear(hidden_dim // 2, 3),
        )

    def forward(self, seg_logits, coord, offset):
        """
        Args:
            seg_logits: (N, num_classes) raw logits from semantic head
            coord: (N, 3) point coordinates
            offset: (B,) batch offset tensor for torch_cluster

        Returns:
            offset_pred: (N, 3) predicted displacement to nearest boundary
        """
        from torch_cluster import knn

        # Soft semantic probabilities (differentiable)
        sem_prob = seg_logits.softmax(dim=-1)  # (N, C)
        feat = torch.cat([sem_prob, coord], dim=-1)  # (N, C+3)

        # Build batch vector from offset for torch_cluster
        batch_vec = torch.zeros(coord.shape[0], dtype=torch.long, device=coord.device)
        for i in range(offset.shape[0]):
            start = 0 if i == 0 else offset[i - 1]
            batch_vec[start:offset[i]] = i

        # KNN on coord (spatial neighbors only)
        # knn returns (2, N*K) tensor: row 0 = query idx, row 1 = neighbor idx
        edge_index = knn(coord, coord, self.k, batch_vec, batch_vec)
        row, col = edge_index[0], edge_index[1]  # row=neighbor, col=center

        # Edge features: [feat_center, feat_neighbor - feat_center, coord_neighbor - coord_center]
        feat_center = feat[col]          # (N*K, C+3)
        feat_neighbor = feat[row]        # (N*K, C+3)
        coord_diff = coord[row] - coord[col]  # (N*K, 3)

        edge_feat = torch.cat([
            feat_center,
            feat_neighbor - feat_center,
            coord_diff,
        ], dim=-1)  # (N*K, 2*(C+3)+3)

        # Shared MLP on edge features
        edge_feat = self.edge_mlp(edge_feat)  # (N*K, hidden_dim)

        # Max-pool over K neighbors per point
        # col contains center point indices, repeated K times each
        N = coord.shape[0]
        # Scatter max: for each center point, take max over its K neighbors
        pooled = torch.zeros(N, edge_feat.shape[1], device=edge_feat.device)
        pooled.scatter_reduce_(0, col.unsqueeze(-1).expand_as(edge_feat), edge_feat, reduce="amax")

        # Offset prediction
        offset_pred = self.offset_head(pooled)  # (N, 3)
        return offset_pred
```

**Key design decisions:**
- Uses `torch_cluster.knn` which is already available in the environment and supports batch vectors
- `softmax(seg_logits)` ensures differentiability (gradients flow back to backbone)
- Edge features `[feat_i, feat_j - feat_i, coord_diff]` directly encode local semantic differences — the core signal for boundary detection
- `scatter_reduce_("amax")` for max-pooling is native PyTorch, no extra dependency
- K=16 default — can be tuned via config

**Validation:** Module instantiates and `forward()` returns shape (N, 3).

```python
class SerialDerivationModel(nn.Module):
    """Semantic segmentation + support head + serial boundary offset derivation.

    Extends SharedBackboneSemanticSupportModel with module g that derives
    boundary offset from semantic logits. The backbone only learns semantic
    features — geometric field is derived serially from semantic output.
    """

    def __init__(
        self,
        num_classes,
        backbone_out_channels,
        offset_k=16,
        offset_hidden_dim=64,
        semantic_adapter_cfg=None,
        boundary_adapter_cfg=None,
        backbone=None,
    ):
        super().__init__()
        # Same components as SharedBackboneSemanticSupportModel
        self.backbone = build_model(backbone)
        self.semantic_adapter = self._build_adapter(backbone_out_channels, semantic_adapter_cfg)
        self.boundary_adapter = self._build_adapter(backbone_out_channels, boundary_adapter_cfg)
        self.semantic_head = SemanticHead(backbone_out_channels, num_classes)
        self.support_head = SupportHead(backbone_out_channels)

        # NEW: serial derivation module g
        self.offset_module = BoundaryOffsetModule(
            num_classes=num_classes,
            k=offset_k,
            hidden_dim=offset_hidden_dim,
        )

    # _build_adapter, _extract_feat: identical to SharedBackboneSemanticSupportModel

    def forward(self, input_dict, return_point=False):
        coord = input_dict["coord"]     # (N, 3) — needed for g
        offset = input_dict["offset"]   # (B,) — needed for batched KNN

        backbone_output = self.backbone(Point(input_dict))
        point, feat = self._extract_feat(backbone_output)

        semantic_feat = self.semantic_adapter(feat)
        boundary_feat = self.boundary_adapter(feat)
        seg_logits = self.semantic_head(semantic_feat)
        support_pred = self.support_head(boundary_feat)

        # Serial derivation: offset from semantic logits (NOT backbone features)
        offset_pred = self.offset_module(seg_logits, coord, offset)

        output = dict(
            seg_logits=seg_logits,
            support_pred=support_pred,
            offset_pred=offset_pred,   # NEW
        )
        if return_point:
            output["point"] = point
        return output
```

**Register** with `@MODELS.register_module()`.

**Key points:**
- `coord` and `offset` extracted from `input_dict` before backbone call (backbone may modify Point structure)
- `offset_pred` is derived from `seg_logits` (serial), not from `feat` (parallel)
- `support_pred` remains from backbone features (CR-C route, separate branch)

**Validation:** Model constructs from config dict and `forward()` returns `{seg_logits, support_pred, offset_pred}`.

**Create** `SerialDerivationLoss(nn.Module)`:

```python
class SerialDerivationLoss(nn.Module):
    """Three-term loss: semantic (CE+Lovasz) + support BCE (CR-C) + offset smooth-L1.

    Extends BoundaryProximityCueLoss with offset supervision on valid points.
    """

    def __init__(self, aux_weight=0.3, offset_weight=1.0):
        super().__init__()
        self.aux_weight = float(aux_weight)
        self.offset_weight = float(offset_weight)
        self.semantic_loss = nn.CrossEntropyLoss(ignore_index=-1)
        self.lovasz_loss = LovaszLoss(mode="multiclass", ignore_index=-1, loss_weight=1.0)

    def forward(
        self,
        seg_logits,      # (N, num_classes)
        support_pred,    # (N, 1) logits
        offset_pred,     # (N, 3) from module g
        segment,         # (N,) class labels
        edge,            # (N, 6) ground truth
        **_extra,
    ):
        segment = segment.reshape(-1).long()
        edge = edge.float()

        # Ground truth
        dir_gt = edge[:, 0:3]                         # (N, 3)
        dist_gt = edge[:, 3:4]                        # (N, 1)
        support_gt = edge[:, 4].clamp(0.0, 1.0)       # (N,)
        valid_gt = edge[:, 5].clamp(0.0, 1.0)         # (N,)
        offset_gt = dir_gt * dist_gt                   # (N, 3)

        # Term 1: Global semantic (CE + Lovasz) — same as CR-C
        loss_ce = self.semantic_loss(seg_logits, segment)
        loss_lovasz = self.lovasz_loss(seg_logits, segment)
        loss_semantic = loss_ce + loss_lovasz

        # Term 2: Confidence-weighted BCE for support — same as CR-C
        support_logit = support_pred[:, 0]
        confidence_weight = torch.where(
            valid_gt > 0.5, support_gt, torch.ones_like(support_gt)
        )
        loss_aux = F.binary_cross_entropy_with_logits(
            support_logit, valid_gt, weight=confidence_weight, reduction="mean"
        )

        # Term 3: Offset smooth-L1 on valid points only
        valid_mask = valid_gt > 0.5  # (N,) bool
        if valid_mask.any():
            loss_offset = F.smooth_l1_loss(
                offset_pred[valid_mask], offset_gt[valid_mask], reduction="mean"
            )
        else:
            loss_offset = offset_pred.sum() * 0.0  # zero but keeps grad graph

        # Total
        total = loss_semantic + self.aux_weight * loss_aux + self.offset_weight * loss_offset

        # Monitoring (detached)
        with torch.no_grad():
            support_prob = torch.sigmoid(support_logit)
            valid_sum = valid_gt.sum().clamp_min(1e-6)
            offset_mae = (offset_pred[valid_mask] - offset_gt[valid_mask]).abs().mean() if valid_mask.any() else torch.tensor(0.0)

        return dict(
            loss=total,
            loss_semantic=loss_semantic,
            loss_ce=loss_ce,
            loss_lovasz=loss_lovasz,
            loss_aux=loss_aux,
            loss_aux_weighted=self.aux_weight * loss_aux,
            loss_offset=loss_offset,
            loss_offset_weighted=self.offset_weight * loss_offset,
            valid_ratio=valid_gt.mean(),
            support_positive_ratio=(support_gt > 1e-3).float().mean(),
            aux_prob_mean=support_prob.mean(),
            aux_prob_boundary_mean=(support_prob * valid_gt).sum() / valid_sum,
            offset_mae=offset_mae,
        )
```

**Key points:**
- Extends CR-C's two terms with a third: offset smooth-L1
- `offset_gt = dir_gt * dist_gt` — computed from existing edge.npy columns, no new ground truth needed
- Only valid points receive offset supervision
- Returns `loss_offset`, `loss_offset_weighted`, and `offset_mae` for monitoring

**Validation:** Loss accepts all required tensors and returns dict with `loss` key.

### Task 6: Update `project/trainer/trainer.py` — route offset_pred to loss

**File:** `project/trainer/trainer.py`

**Change `_build_loss_inputs()`** — add a new branch for `offset_pred`:

After the existing `elif "support_pred" in output and "edge" in batch:` block, the logic needs to handle the case where both `support_pred` AND `offset_pred` are present. Since `SerialDerivationModel` outputs both, and `SerialDerivationLoss` expects both:

**New routing logic:**
```python
@staticmethod
def _build_loss_inputs(output: dict, batch: dict) -> dict:
    kwargs = dict(seg_logits=output["seg_logits"], segment=batch["segment"])

    if "edge_pred" in output and "edge" in batch:
        # Full boundary model (SharedBackboneSemanticBoundaryModel)
        kwargs["edge_pred"] = output["edge_pred"]
        kwargs["edge"] = batch["edge"]
        if "coord" in batch:
            kwargs["coord"] = batch["coord"]
        if "offset" in batch:
            kwargs["offset"] = batch["offset"]
        if "support_id" in batch:
            kwargs["support_id"] = batch["support_id"]
    elif "support_pred" in output and "edge" in batch:
        # Support-only or serial derivation model
        kwargs["support_pred"] = output["support_pred"]
        kwargs["edge"] = batch["edge"]
        # Serial derivation: also pass offset_pred if present
        if "offset_pred" in output:
            kwargs["offset_pred"] = output["offset_pred"]

    return kwargs
```

This is a minimal change — just add `offset_pred` passthrough inside the existing `support_pred` branch. `SerialDerivationLoss.forward()` accepts `offset_pred` as a kwarg; `BoundaryProximityCueLoss` ignores it via `**_extra`.

### Task 9: Smoke validation

**Scope:** Verify the full CR-D pipeline end-to-end without running a full training.

**Method:** Run existing pipeline check pattern (or manual 1-batch test):
1. Dataset construction + DataLoader iteration (1 batch)
2. Model forward pass — verify output keys: `{seg_logits, support_pred, offset_pred}`
3. Verify `offset_pred.shape == (N, 3)`
4. Loss computation — verify all loss terms are finite
5. Backward pass + optimizer step — verify no gradient errors
6. Verify gradients flow from offset loss through g through softmax back to backbone:
   - `model.backbone.parameters()` should have non-zero `.grad` from offset loss alone

**Definition of Done:** All checks pass. Pipeline is ready for full training run.

## Risks

1. **KNN computation cost**: `torch_cluster.knn` with K=16 on ~100k points per batch. Should be fast on GPU but needs measurement. If slow, K can be reduced.
2. **scatter_reduce_ compatibility**: `scatter_reduce_` with `reduce="amax"` requires PyTorch >= 1.12. Should be fine in current environment but verify.
3. **Memory**: g adds (N×K×25) edge features + (N×64) pooled features. For N=100k, K=16: ~160MB for edge features in float32. Should fit in GPU memory alongside PTv3 backbone.
4. **coord alignment**: `input_dict["coord"]` must have the same N as `seg_logits` after backbone processing. Verify this — PTv3's decoder should restore to original resolution.
