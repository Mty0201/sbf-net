```python
class SemanticOnlyLoss(nn.Module):
    def __init__(self):
        super().__init__()
        self.semantic_loss = nn.CrossEntropyLoss(ignore_index=-1)
        self.lovasz_loss = LovaszLoss(mode="multiclass", ignore_index=-1, loss_weight=1.0)

    def forward(self, seg_logits, segment) -> dict[str, torch.Tensor]:
        loss_semantic = self.semantic_loss(seg_logits, segment)
        loss_lovasz = self.lovasz_loss(seg_logits, segment)
        total_loss = loss_semantic + loss_lovasz
        return dict(loss=total_loss, loss_semantic=loss_semantic, loss_lovasz=loss_lovasz)
```

**Required forward() signature:**

**Exact loss math — three terms:**

1. **Global semantic loss (loss_semantic):** Same as SemanticOnlyLoss — `CrossEntropyLoss(ignore_index=-1)` on `seg_logits` vs `segment` plus `LovaszLoss(mode="multiclass", ignore_index=-1)` on same. Lovasz stays global (not support-weighted). `loss_semantic = loss_ce + loss_lovasz`.

2. **Support supervision (loss_support):** Binary cross-entropy with logits on `support_pred[:, 0]` against `support_gt * valid_gt`, masked by `valid_gt`. Exact code:
```python
support_logit = support_pred[:, 0]
support_gt = edge[:, 4].float().clamp(0.0, 1.0)
valid_gt = edge[:, 5].float().clamp(0.0, 1.0)
support_target = support_gt * valid_gt
loss_support_raw = F.binary_cross_entropy_with_logits(
    support_logit, support_target, reduction="none"
)
# Weighted mean over valid points only
valid_sum = valid_gt.sum().clamp_min(1e-6)
loss_support = (loss_support_raw * valid_gt).sum() / valid_sum
```
When there are zero valid boundary points in a batch (`valid_gt.sum() == 0`), `loss_support` is effectively 0 because `valid_gt` is all zeros and the numerator is zero. The `clamp_min(1e-6)` prevents division by zero.

3. **Support-guided semantic focus (loss_focus):** Pointwise CE weighted by boundary proximity. Exact code:
```python
point_ce = F.cross_entropy(seg_logits, segment.reshape(-1).long(), ignore_index=-1, reduction="none")
focus_weight = 1.0 + self.focus_lambda * (support_gt * valid_gt).pow(self.focus_gamma)
# Exclude ignore-index points from weighting
valid_sem = (segment.reshape(-1).long() != -1)
focus_weight = focus_weight * valid_sem.float()
loss_focus = (point_ce * focus_weight).sum() / focus_weight.sum().clamp_min(1e-6)
```
When there are zero valid boundary points, `support_gt * valid_gt` is all zeros, so `focus_weight = 1.0 * valid_sem` and the focus term degrades gracefully to unweighted CE over valid semantic points.

**Total loss:**
```python
total_loss = loss_semantic + self.support_loss_weight * loss_support + self.focus_loss_weight * loss_focus
```

**Return dict keys (all torch.Tensor):**
```python
return dict(
    loss=total_loss,
    loss_semantic=loss_semantic,
    loss_ce=loss_ce,
    loss_lovasz=loss_lovasz,
    loss_support=loss_support,
    loss_focus=loss_focus,
    valid_ratio=valid_gt.mean(),
    support_positive_ratio=(support_gt > 1e-3).float().mean(),
)
```

**Metrics contract — what the evaluator reports:**

1. **Global semantic metrics (same as SemanticEvaluator):**
   - `val_mIoU`, `val_mAcc`, `val_allAcc` — computed identically to `SemanticEvaluator`
   - `semantic_intersection`, `semantic_union`, `semantic_target` — per-class accumulators
   - `semantic_iou_per_class`, `semantic_acc_per_class` — per-class breakdowns
   - `val_loss_semantic` — from internal `SemanticOnlyLoss` call

2. **Boundary-region semantic metrics (NEW):**
   - `val_boundary_mIoU` — mIoU computed only over points where `support_gt >= boundary_metric_threshold`
   - `val_boundary_mAcc` — mAcc computed only over boundary-region points
   - `boundary_point_ratio` — fraction of points in the boundary region

3. **Support metrics (support-specific, NOT legacy dir/dist field metrics):**
   - `support_bce` — binary cross-entropy of `sigmoid(support_pred[:, 0])` vs `support_gt * valid_gt`, averaged over valid points
   - `support_cover` — Tversky overlap (alpha=0.3, beta=0.7) of `sigmoid(support_pred[:, 0])` vs `(valid_gt > 0.5).float()`
   - `valid_ratio` — mean of `valid_gt`
   - `support_positive_ratio` — fraction of points where `support_gt > 1e-3`

**Metrics that DISAPPEAR from the active route (vs SemanticBoundaryEvaluator):**
- NO `dir_cosine`, `dist_error`, `dist_error_scaled`
- NO `val_loss_edge`, `val_loss_dir`, `val_loss_dist`
- NO `dir_valid_ratio`, `dist_gt_valid_mean`

**Implementation pattern:**
- Reuse the per-class intersection/union/target loop from `SemanticEvaluator`.
- For boundary-region metrics, create a boolean mask `boundary_mask = support_gt >= self.boundary_metric_threshold` where `support_gt = edge[:, 4].float().clamp(0.0, 1.0)`, then compute per-class intersection/union/target only over masked points.
- Use `SemanticOnlyLoss` internally for `val_loss_semantic`, same as `SemanticEvaluator`.
- Include the same `_safe_zero` and `_safe_div` helpers as `SemanticEvaluator`.
