**Part A: Create test_density_adaptive.py**

Create a new test file `data_pre/bf_edge_v3/tests/test_density_adaptive.py` that programmatically verifies DEN-02 and DEN-03. This test runs the Phase 5 pipeline in-memory on the 020101 and 020102 scenes and checks the density gap.

The test should:

1. Import the density analysis functions from `diagnose_net01.py` (compute_scene_knn_distances, assign_density_buckets, analyze_stage2).

2. For each scene (020101 at `samples/training/020101`, 020102 at `samples/validation/020102`):
   a. Load boundary_centers.npz and coord.npy from the scene directory
   b. Run `cluster_boundary_centers()` with default `Stage2Config()` (Phase 5 defaults)
   c. Compute scene-wide kNN density (k=10) on scene points
   d. Compute center-level density buckets using P25/P75 of center densities
   e. Call `analyze_stage2()` to get per-bucket survival rates
   f. Compute gap = dense_rate - sparse_rate
   g. Assert gap < 0.05 (DEN-02)
   h. Assert dense_rate >= 0.99 (DEN-03 — dense-region non-regression)

3. Mark both scene tests with `pytest.mark.skipif` if the scene data does not exist on disk (same pattern as test_equivalence.py).

4. Add a unit test for the density-conditional denoise behavior using synthetic data:
   - Create two clusters: one dense (tight spacing ~0.01), one sparse (loose spacing ~0.05)
   - Run `cluster_boundary_centers()` with default config
   - Verify that the sparse cluster was NOT denoised (all points survive)
   - Verify that the dense cluster WAS denoised (or at least the denoise function was invoked — check via meta dict's num_denoise_skipped)
