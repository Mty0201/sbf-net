**Structure:**

**Header:** Title, purpose ("Documents implicit cross-stage behavioral contracts -- where one stage assumes specific properties of another stage's output").

**Per-contract sections (one per coupling identified in RESEARCH.md Section 3, plus any additional ones found during audit):**

**## Contract: Stage 1 -> Stage 2**
- NPZ fields consumed: center_coord(M,3), center_tangent(M,3), semantic_pair(M,2), confidence(M,)
- Semantic invariants: Arrays are aligned by index (row i of each array describes the same boundary center)
- Assumption: M > 0 (Stage 2 groups by semantic_pair, empty input produces empty output)

**## Contract: Stage 1 -> Stage 3 (read-back)**
- NPZ fields consumed: All center fields via center_index from Stage 2
- Critical coupling: Stage 3 uses center_index values from Stage 2 to index into Stage 1's arrays. center_index[k] must be a valid index into Stage 1's arrays.
- Preservation requirement: Stage 1 arrays must never be reordered or filtered between Stage 1 output and Stage 3 input
- Risk: High if intermediate format changes; Low as long as NPZ serialization preserved

**## Contract: Stage 2 -> Stage 3 (trigger flag semantics)**
- NPZ fields consumed: center_index(K,), cluster_id(K,), semantic_pair(C,2), cluster_trigger_flag(C,), cluster_size(C,)
- Trigger semantics: cluster_trigger_flag > 0 means "cluster likely contains multiple edge branches -- Stage 3 should use direction-based regrouping (regroup_trigger_cluster) instead of direct fitting"
- Hidden assumption: If trigger flag is set, the cluster's tangent vectors are meaningful enough for direction-based grouping
- Risk: If Stage 2 trigger criteria change, Stage 3 regrouping may get clusters it cannot handle

**## Contract: Stage 3 -> Stage 4 (support schema)**
- NPZ fields consumed: support_id(S,), semantic_pair(S,2), segment_offset(S,), segment_length(S,), segment_start(T,3), segment_end(T,3), line_start(S,3), line_end(S,3)
- Invariant: Every support has EITHER valid segments (segment_length > 0) OR valid line endpoints (line_start/line_end). No validation enforces this.
- Risk: If Stage 3 produces a support with neither, Stage 4 silently produces infinite distances

**## Contract: Scene -> Stage 4 (direct read)**
- Fields consumed: coord.npy, segment.npy (skips Stages 2-3)
- No intermediate dependency -- reads raw scene data

**## Contract: Stage 3 internal (spacing-adaptive thresholds)**
- All trigger regrouping thresholds computed as scale_factor * local_spacing
- local_spacing from estimate_local_spacing(coords, k=6)
- Already density-adaptive at cluster level, separate from Stage 2 fixed-eps
- Edge case: clusters with fewer than k=6 points get clamped spacing via EPS

**## Contract: In-memory bypass (build_support_dataset_v3.py)**
- Runs Stages 1-3 in-memory without intermediate NPZ I/O
- Assumption: In-memory path produces identical results to per-stage NPZ path (true for float32/int32 numpy arrays)
- Coupling: Any change to core function signatures must be reflected in BOTH per-scene scripts AND build_support_dataset_v3.py
- Lists the specific function calls made in the in-memory path vs the per-scene path

**NPZ Schema Summary Table:** A single table listing every NPZ file, all fields, shapes, dtypes, and which stages produce/consume each.
