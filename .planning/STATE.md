### CR-SDE Architecture Redesign (2026-04-12)

**Problem:** CR-SD's SubtractiveDecoupling creates two divergent feature streams, but without a fusion mechanism the two branches' gradients conflict in the backbone at late training â€” semantic loss is small (gradient small) while margin loss plateaus (gradient large), causing the boundary branch to dominate backbone updates.

**Analysis of BFANet original:** BFANet's official ES Attention (K=1) is degenerate â€” softmax over a single token â‰¡ 1.0, so the fused query does nothing at inference. Our gv4 CrossStreamFusionAttention (K=1024 patches) is a structural fix that makes the fusion non-trivial.

**Redesign (commit 2801ad6):** Replaced GatedSegRefiner (single-direction margâ†’seg scalar gate) with CrossStreamFusionAttention (gv4): BFANet-style Q-fusion bidirectional cross-attention. Both streams exchange information through a shared fused query attending separately to each stream's K/V. Zero-init output projections guarantee step-0 identity (CR-SD equivalence at init).

**v1/v2 dual supervision added:** Following BFANet's dual-stage loss design, added v1 prediction heads (pre-fusion) alongside v2 heads (post-fusion). Both stages supervised with identical loss: s_weight 5Ã— CE + LovÃ¡sz + BCE + Dice. v1:v2 weight = 1:1. v1 supervision anchors the decoupling MLP gradients directly, preventing the fusion module from mixing the streams back together.

**Full CR-SDE architecture:**
1. PTv3 backbone â†’ decoder features
2. SubtractiveDecoupling â†’ seg_feat, marg_feat (divergent streams)
3. v1 heads: seg_head_v1(seg_feat), marg_head_v1(marg_feat) â†’ v1 logits
4. CrossStreamFusionAttention(seg_feat, marg_feat) â†’ seg_feat_v2, marg_feat_v2
5. v2 heads: seg_head_v2(seg_feat_v2), marg_head_v2(marg_feat_v2) â†’ v2 logits (final output)

**Loss:** L = (CE_w + LovÃ¡sz + auxÃ—(BCE+Dice))_v2 + (CE_w + LovÃ¡sz + auxÃ—(BCE+Dice))_v1. CRSDLoss accepts optional `seg_logits_v1`/`marg_logits_v1`; CR-SD (no fusion) passes None and v1 terms are skipped.
