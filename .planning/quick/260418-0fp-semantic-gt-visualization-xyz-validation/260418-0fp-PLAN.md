# Quick Task 260418-0fp: Semantic GT Visualization (Validation Sets)

## Goal

Generate per-point XYZRGB point clouds for every validation sample across three datasets (BF / S3DIS / ZAHA), where RGB encodes the semantic ground-truth class. Output under a sibling folder so the originals stay untouched, plus a single legend file mapping class→color.

## Inputs (per sample)

| Dataset | Root                                       | Split         | Classes | Files used               |
|---------|--------------------------------------------|---------------|---------|--------------------------|
| BF      | /home/mty0201/data/BF_edge_chunk_npy       | validation/*  | 8       | coord.npy, segment.npy   |
| S3DIS   | /home/mty0201/data/s3dis                   | validation/*  | 13      | coord.npy, segment.npy   |
| ZAHA    | /home/mty0201/data/ZAHA_chunked            | validation/*  | 5       | coord.npy, segment.npy   |

Class names (fixed tables):
- BF: balustrade, balcony, advboard, wall, eave, column, window, clutter
- S3DIS: ceiling, floor, wall, beam, column, window, door, table, chair, sofa, bookcase, board, clutter
- ZAHA: floor, decoration, structural, opening, other_el

## Outputs

Each dataset gets a sibling `..._gt_viz/` folder next to its data root. Label value -1 (ignore) renders as black `(0,0,0)` and is noted in the legend.

```
/home/mty0201/data/BF_edge_chunk_npy_gt_viz/validation/<sample>.xyz
/home/mty0201/data/s3dis_gt_viz/validation/<sample>.xyz
/home/mty0201/data/ZAHA_chunked_gt_viz/validation/<sample>.xyz
/home/mty0201/data/<dataset>_gt_viz/legend.txt
```

Each `.xyz` line: `x y z R G B` (float coords, uint8 RGB [0,255], space-separated). `legend.txt` format:
```
# Dataset: <name>
# label  class_name  R  G  B
0  ceiling  230  25  75
1  floor    60   180 75
...
-1 ignore  0    0   0
```

## Task

### 1. `scripts/viz/gt_color_export.py`

Single standalone Python script. CLI:
```
python scripts/viz/gt_color_export.py \
    --dataset {bf,s3dis,zaha,all} \
    [--data-root PATH] [--out-root PATH] \
    [--split validation] [--limit N] [--overwrite]
```

Behavior:
- Hardcode default roots + class-name tables above. `--dataset all` loops all three.
- Use a distinct-color palette (hardcoded 32 colors from Glasbey / Kelly's 22 + safe extras) so classes render clearly.
- Palette mapping is **stable and shared by class position** (class 0 always color 0). Legend is written once per dataset with the truncated palette for its class count.
- For each sample dir under `<data_root>/<split>/`, load `coord.npy` (float32 Nx3) and `segment.npy` (int Nx1 or N). Map labels → RGB. Skip sample with `print` warning if either file missing.
- Write `<out_root>/<split>/<sample>.xyz` using `np.savetxt` with `fmt=["%.4f","%.4f","%.4f","%d","%d","%d"]`. Use `--overwrite` to rewrite existing; otherwise skip existing.
- Write `legend.txt` at `<out_root>/legend.txt`.
- Print final summary: samples processed / skipped per dataset.

**Files to create:**
- `scripts/viz/__init__.py` (empty, if `scripts/viz/` doesn't exist)
- `scripts/viz/gt_color_export.py`

**Action:** Write the script using numpy only (no torch dependency). Handle segment shape `(N,)` or `(N,1)`. Clamp invalid labels (>= num_classes) to -1 / ignore color.

**Verify:** Run `python scripts/viz/gt_color_export.py --dataset zaha --limit 1` end-to-end; confirm one `.xyz` produced with correct shape and RGB values match legend.

**Done:** Script exists, smoke-run on 1 ZAHA sample produces `.xyz` + `legend.txt`; committed.
